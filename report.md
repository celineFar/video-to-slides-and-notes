## 1. Project Overview

**Purpose:** A YouTube video-to-PDF slide generator. It downloads videos, extracts transcripts, captures screenshot frames at intervals, matches each frame to its transcript segment, and produces PDF slide decks.

**Tech stack:** Python 3, OpenCV, yt-dlp, FFmpeg, ReportLab, PyPDF2, LangChain, Pillow, tqdm, PyYAML.

**Files:**

| File | Lines | Role |
|------|-------|------|
| `main.py` | 562 | Pipeline orchestrator — config loading, chunking, screenshot extraction, transcript matching, PDF assembly |
| `youtube_screenshot_script.py` | 790 | Frame extraction engine — video download, quality scoring, filtering, parallel processing |
| `pdf_api.py` | 127 | PDF generation — image-to-PDF conversion, text wrapping, merging |
| `transcript_api.py` | 60 | Transcript fetching — YouTube transcript download and chunking |
| `chunking_utils.py` | 128 | Video segmentation — FFmpeg-based splitting with fast/safe fallback |
| `config.yaml` | 37 | Runtime configuration |
| `requirements.txt` | 27 | Dependencies (no version pins) |
| `scripts/upload.sh` | 91 | Upload videos/transcripts to a remote server |
| `scripts/download_pdfs.sh` | 66 | Download generated PDFs from a remote server |

## 2. Architecture & Structure

### Module Dependency Graph

```
main.py
├── chunking_utils.py    (chunk)
├── youtube_screenshot_script.py  (run_extraction, sanitize_filename, download_video)
├── pdf_api.py           (image_to_pdf, merge_pdfs)
└── transcript_api.py    (get_chunked_transcript)
```

### Pipeline Flow

```
config.yaml / CLI args
        │
        ▼
    main.main()
        │
        ├─ 1. Resolve video source (download or local path)
        │     └─ youtube_screenshot_script.download_video()
        │
        ├─ 2. Load or fetch transcript
        │     ├─ pickle.load() for local transcripts
        │     └─ transcript_api.get_chunked_transcript() for YouTube URLs
        │
        ├─ 3. Chunk video at timestamps (optional)
        │     └─ chunking_utils.chunk()
        │         ├─ fast_split() → ffmpeg segment muxer (lossless)
        │         └─ safe_transcode() → H.264 re-encode (fallback)
        │
        ├─ 4. For each chunk:
        │     ├─ Extract screenshots (with pickle caching)
        │     │   └─ youtube_screenshot_script.run_extraction()
        │     │       └─ extract_frames() → process_frame() per frame
        │     │
        │     ├─ Match each screenshot timestamp to a transcript chunk
        │     │   └─ find_matching_chunk()
        │     │
        │     ├─ Generate one PDF page per screenshot
        │     │   └─ pdf_api.image_to_pdf()
        │     │
        │     └─ Merge pages into final PDF
        │         └─ pdf_api.merge_pdfs()
        │
        └─ 5. Write debug logs
```

## 3. Data Flow

**Transcript data** flows as `List[Tuple[float, float, str]]` — a list of `(start_seconds, end_seconds, text)` tuples. This representation is used everywhere but is never formalized into a named type or dataclass.

**Screenshot data** flows as `List[Tuple[float, str]]` — a list of `(timestamp, image_path)` tuples. These are generated by `get_timestamped_frames()` by parsing filenames like `frame_000042_q85_b120.png`.

**Caching** uses pickle files in `extracted_frames_cache/` keyed by video name, interval, and chunk start time.

## 4. Design Culprits

### 4.1 Critical Bugs

#### Duplicate `download_video()` definition (youtube_screenshot_script.py:27 and :60)

Two functions with the same name exist in the same file. Python silently uses the second one, making the first one dead code. The second definition references an undefined `logger` variable on line 120, which will cause a `NameError` at runtime if a download retry is triggered.

```python
# Line 120 — logger is never imported or defined
logger.warning("Download attempt %s/%s failed: %s", attempt, max_retries, exc)
```

#### `VIDEO_INTERVAL` referenced but never defined (main.py:387, :419)

Inside `create_slides()`, the code references `VIDEO_INTERVAL` which does not exist anywhere in the codebase. This function would crash if called.

#### `import shlex` in the middle of the file (youtube_screenshot_script.py:240)

An import statement appears between function definitions instead of at the top of the file. While Python allows this, it signals disorganized code and violates PEP 8.

### 4.2 Dead Code

| Location | What | Why it's dead |
|----------|------|---------------|
| `main.py:20-23` | `SCREENSHOT_INTERVAL`, `TIMESTAMPS`, `VIDEO_PATH`, `TRANSCRIPT_INTERVAL` module-level constants | Overridden by `config.yaml` values in `main()`. Only used as default arguments in legacy functions. |
| `main.py:345-423` | `create_slides()` function | Never called. The `main()` function (line 437) is the actual entry point. Contains the undefined `VIDEO_INTERVAL` reference. |
| `main.py:10` | `from collections import defaultdict` | Never used anywhere in the file. |
| `youtube_screenshot_script.py:27-56` | First `download_video()` definition | Overwritten by the second definition on line 60. |
| `chunking_utils.py:117-127` | `main()` with hardcoded file paths | Test harness left in production code. |

### 4.3 Architectural Problems

#### No separation between I/O and logic

Every function mixes computation with file system operations, `print()` calls, and `os.makedirs()`. This makes unit testing impossible without mocking the entire OS layer. For example, `extract_screenshots_cached()` handles cache lookup, file I/O, frame extraction, and pickle serialization all in one function.

#### No error propagation strategy

Errors are handled inconsistently:
- Some functions return `None` on failure (`download_if_needed`, `download_timestamped_transcript`)
- Some call `sys.exit(1)` (`extract_frames`)
- Some raise exceptions (`chunk`, second `download_video`)
- Some silently continue (`find_matching_chunk` returns `""`)

There is no consistent pattern. A caller cannot reliably distinguish "no transcript exists" from "network error" from "parse failure."

#### God function: `extract_frames()` (youtube_screenshot_script.py:347-474)

This 128-line function handles:
- Video file opening and validation
- Frame interval calculation for 4 different methods
- FFmpeg subprocess calls for keyframe extraction
- Scene detection library integration
- Frame loading with progress bars
- Parallel vs sequential processing orchestration
- Resume logic with JSON progress files
- Statistics tracking

Each of these should be a separate function or class method.

#### Tight coupling via filename conventions

The system relies on filenames like `frame_000042_q85_b120.png` to carry metadata (frame index, quality score, blur score). `get_timestamped_frames()` parses these with a regex to reconstruct timestamps. If the naming scheme in `process_frame()` changes, the entire matching pipeline breaks silently.

#### Pickle for caching and transcript storage

Using pickle for caching (`extracted_frames_cache/*.pkl`) and transcript storage is fragile:
- Pickle files are not human-readable or debuggable
- They are tied to the exact Python class/module structure
- They pose a security risk if files come from untrusted sources (arbitrary code execution on load)
- No schema validation — a corrupted or outdated pickle silently produces wrong data

#### No configuration validation

`config.yaml` is loaded with `yaml.safe_load()` and its keys are accessed with `.get()` and dictionary indexing. There is no validation that required keys exist, that values are the right type, or that paths actually point to existing files. A typo like `screensht_interval` would be silently ignored.

#### `os.system()` for FFmpeg commands (youtube_screenshot_script.py:373)

```python
os.system(ffmpeg_command)
```

This is a shell injection vulnerability. The `video_path` variable is interpolated directly into a shell command string via f-string. A filename containing `; rm -rf /` would execute arbitrary commands. The rest of the codebase correctly uses `subprocess.run()` with argument lists.

### 4.4 Missing Infrastructure

| What's missing | Impact |
|----------------|--------|
| **Tests** | Zero test files. No way to verify correctness or catch regressions. |
| **Logging** | All output uses `print()`. No log levels, no way to control verbosity per-module, no structured output for production. |
| **Type hints** | Partial coverage. `youtube_screenshot_script.py` has almost none. Return types missing on most functions. |
| **Docstrings** | Only `main.py` and `transcript_api.py` have meaningful docstrings. |
| **Version pins** | `requirements.txt` has no version constraints. `yt-dlp` and `youtube-transcript-api` break frequently. |
| **Cleanup on failure** | Partial PDFs, temp files, and incomplete cache entries are not cleaned up on error. |

| `youtube_screenshot_script.py` | "script" is redundant, "youtube" is misleading (it handles local files too) | `frame_extractor.py` |

### 6.2 Typed Data Models

Replace raw tuples with dataclasses so the code is self-documenting:

```python
from dataclasses import dataclass

@dataclass
class TranscriptChunk:
    start: float
    end: float
    text: str

@dataclass
class TimestampedFrame:
    timestamp: float
    file_path: str
    quality_score: float
    blur_score: float

@dataclass
class FrameProcessingConfig:
    quality_threshold: float = 12.0
    blur_threshold: float = 10.0
    detect_watermarks: bool = False
    watermark_threshold: float = 0.8
    use_png: bool = False
    use_gpu: bool = False
    gradfun: bool = False
    deblock: bool = False
    deband: bool = False
```

This replaces the 13-element tuple in `process_frame()` and makes every function signature readable.

### 6.3 Configuration with Validation

```python
from pydantic import BaseModel, Field, validator
from pathlib import Path

class PipelineConfig(BaseModel):
    video_path: str
    video_title: str
    screenshot_interval: float = Field(gt=0, default=4.5)
    transcript_interval: int = Field(gt=0, default=9)
    max_resolution: int | None = None
    split_timestamps: list[str] = []
    transcript_path: Path | None = None
    screenshots_dir: str = "screenshots"
    verbose: bool = False

    @validator("transcript_path")
    def transcript_must_exist(cls, v):
        if v is not None and not v.exists():
            raise ValueError(f"Transcript file not found: {v}")
        return v
```

This catches configuration errors immediately at startup, not 10 minutes into a processing run.

from src.services.frame_extractor import calculate_quality_score

def test_quality_score_black_frame():

### 6.8 Key Refactoring Priorities (Ordered)

If refactoring incrementally rather than rewriting:

1. **Delete dead code** — Remove `create_slides()`, the first `download_video()`, unused constants, unused imports. This is zero-risk and immediately reduces confusion.

2. **Fix the `logger` bug** — Add `import logging; logger = logging.getLogger(__name__)` to `youtube_screenshot_script.py`. Or replace with `print()` to match the rest of the codebase until a full logging migration.

3. **Fix `os.system()` injection** — Replace the `os.system(ffmpeg_command)` call on line 373 with `subprocess.run()` using a list of arguments.

4. **Add version pins** to `requirements.txt`.

5. **Introduce dataclasses** for `TranscriptChunk` and frame processing args. This can be done file-by-file without breaking anything.

6. **Add config validation** — Even a simple function that checks required keys and types at startup prevents silent failures.

7. **Replace `print()` with `logging`** — Module by module, starting with `main.py`.

8. **Break up `extract_frames()`** — Split into `load_frames()`, `process_and_filter_frames()`, and `save_progress()`.

9. **Add tests** — Start with pure functions like `calculate_quality_score()`, `parse_time()`, `wrap_text()`, and `find_matching_chunk()`.

10. **Replace pickle caching** with JSON for transcript data and a proper cache directory structure for frames.
